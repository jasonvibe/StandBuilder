# StandBuilder

一套全面的体系管理与生成工具，支持浏览与导入现有体系资产，快速生成定制化体系文件，并提供在线查看、模块下载和整体打包功能。

## ✨ 功能特点

- **Excel资产导入与识别**
  - 支持上传Excel文件（.xls, .xlsx），系统自动识别文件名中的模块名称、适用行业和项目类型。
  - 支持批量管理已上传的体系文件。

- **多源体系生成与去重**
  - **多源抽取**：根据用户选择的模块，自动从所有已上传的同类文件中抽取数据。
  - **智能合并**：将多个来源的标准条目合并，并基于内容签名进行**自动去重**，确保生成结果干净无冗余。
  - **来源追溯**：生成的体系文件会清晰标注数据来源（如：来源：深铁置业 (1801项), 星河地产 (577项)）。

- **精细化筛选与定制**
  - **行业与类型筛选**：支持按“适用行业”（如住宅、商业）和“项目类型”（如精装修、新建工程）进行数据过滤。
  - **智能关键词匹配**：支持输入“特殊要求”（如：只看土建和电气），系统通过全字段模糊匹配和智能停用词过滤，精准筛选出符合需求的内容。
  - **通用内容兼容**：智能识别“通用”、“General”等标记，确保通用标准不被误过滤。

- **文件下载与导出**
  - 支持在线预览生成的体系内容。
  - 提供一键打包下载功能，将生成的体系文件导出为 ZIP 包（包含整理好的 Excel 文件）。

- **数据安全与隐私**
  - 纯前端处理逻辑（演示版），数据在本地内存中流转。
  - 项目配置了严格的 `.gitignore` 规则，确保 Excel 源文件、数据库文件和生成的敏感数据不会被提交到版本控制系统中。

## 📦 技术栈

- **前端框架**：React 19 + TypeScript
- **构建工具**：Vite
- **UI 组件**：Tailwind CSS + Lucide React + Radix UI
- **数据处理**：SheetJS (xlsx) 用于 Excel 读写
- **文件打包**：JSZip + FileSaver

## 🚀 安装与启动

### 1. 克隆项目
```bash
git clone https://github.com/yourusername/StandBuilder.git
cd StandBuilder/app
```

### 2. 安装依赖
```bash
npm install
```

### 3. 启动开发服务器
```bash
npm run dev
```
访问终端输出的本地地址（通常是 http://localhost:5173）即可开始使用。

## 📖 使用指南

1.  **上传文件**：
    *   点击首页的“上传Excel文件”区域。
    *   选择包含标准体系的 Excel 文件。
    *   系统会自动解析元数据，您也可以手动调整“所属模块”和“适用行业”。

2.  **生成体系**：
    *   点击右下角的“生成体系文件”按钮。
    *   选择您需要的**适用行业**（如：住宅）。
    *   勾选需要生成的**模块**（如：工序验收）。
    *   （可选）开启“AI 智能推荐”并输入**特殊要求**（如：“我只要土建和机电的内容”）。

3.  **查看与下载**：
    *   在生成结果页面，查看合并后的体系详情。
    *   确认“匹配逻辑”和“数据来源”是否符合预期。
    *   点击右上角的“打包下载新体系”按钮导出文件。
